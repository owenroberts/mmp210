A loop defines a set of commands to execute more than once.

JavaScript uses do, while and for.

Loops look like other control statements.

if (condition) { 
	// condtional code 
}

A loop is sort of like copying and pasting the same piece of code multiple times.

while loop

A while statements executes as long as the conditional statement is true.

var i = 0; if (i < 5) { i++; console.log(i); }
var i = 0; while (i < 5) { i++; console.log(i); }

The while loop is useful but it can be a little dangerous to write. This works because the loop increments i and then evaluates it, so the same value is being changed and used to determine when to stop the loop. If we wanted to do something a bunch of times other than increment x, say something like console.log("banana"); we might write a loop that never reaches it's ending condition. That will cause a stack overflow in the browser, meaning the the loop will never stop executing so our program will never finish running.

for loop

We will most often use the for loop, so that's where we'll start today. The for loop works the same way as the while loop, but it is written in a syntax that includes the beginning, update and conditional expressions in one line.

for (var i = 0; i < 5; i++) { console.log("This is the " + i + " loop."); }

The for keyword indicates the beginning of the for loop.

Inside the parentheses there are three statements that control the duration of the loop.

The first statement is the initial condition, where we start counting.

var i = 0;

The second statement creates a condition.

i < 5;

As long as this condition evaluates to true, the loop will continue.

The third statement is the update. This is how the counting variable is changed after each loop.

i++
for (initialization; conditional; iteration) {
	loop body;
}

Loops are an efficient way to execute actions on a list or array of data.

Most of the time we start counting at 0:

var i = 0;

But our initial condition could be anything.

for (var i = 1; i < 6; i++) { console.log("This is the " + i + " loop."); }

Likewise, we can change the conditional and iteration statements.

for (var i = 0; i < 100; i += 10) { console.log("This is the " + i + " loop."); }
for (var i = 10; i > 0; i--) { console.log("This is the " + i + " loop."); }

With a for loop we don't need to worry about forgetting to increment our index value because it's written into the statement. So we can do whatever we want in the loop block.

for (var i = 0; i < 5; i++) { console.log("I like bananas."); }

Scope

Note on scope: in a for loop using the var declaration to initialize i creates a variable that still exists after the foor loop is done.

for (var i = 0; i < 5; i++) { console.log("loop", i); } console.log("global", i);

We can run a new for loop, redeclaring i or resetting it to 0.

// this is ok for (var i = 0; i < 5; i++) { console.log(i, "bananas"); } for (var i = 0; i < 5; i++) { console.log(i, "apples"); }
// this also works for (var i = 0; i < 5; i++) { console.log(i, "bananas"); } for (i = 0; i < 5; i++) { console.log(i, "apples"); }
